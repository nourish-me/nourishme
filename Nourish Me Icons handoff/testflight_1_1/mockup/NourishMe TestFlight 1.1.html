Icons · Implementation

Alle 8 Icons sind als SVG-Source im handoff/icons/svg/ Ordner. Base-Größe ist 24px, skaliert sauber auf 48 / 72 (2× / 3×). PNG-Export folgt auf Approval — sag mir, falls sich noch Geometrien ändern sollen.

// Naming · pubspec.yaml asset map · alle Icons in einem Ordner assets: - assets/icons/ic_nm_pregnancy.svg - assets/icons/ic_nm_nursing.svg - assets/icons/ic_nm_pumping.svg - assets/icons/ic_nm_multiples.svg - assets/icons/ic_nm_meal.svg - assets/icons/ic_nm_food_safety.svg - assets/icons/ic_nm_coach.svg - assets/icons/ic_nm_journal.svg // Render mit flutter_svg · token IDs statt Hex SvgPicture.asset( 'assets/icons/ic_nm_pregnancy.svg', width: 24, colorFilter: ColorFilter.mode(NMColors.pine, BlendMode.srcIn), );

Was offen ist · vor Iteration 2

  • Schreibweise „Tagesziel". Aktuell deutsche Schreibweise mit Punkt (2.880 kcal). Falls ihr im Code lieber Komma-getrennt rendert, sag Bescheid.
  • Aktivitätslevel-Multiplikatoren. Ich habe 1.2 / 1.4 / 1.6 / 1.8 angesetzt — DGE-Standard. Falls eure Berechnung andere Faktoren nutzt, ist es ein 4-Zeilen-Tausch.
  • Multi-State-Default. Phase-Screen zeigt 2 Cards selected (schwanger + stillend) als Demo. Im echten Flow sind alle initial leer.
  • Food Safety Logo-Variante. Plum-Diamond → falls du lieber den Bowl-Stil ohne Diamant willst (z.B. pine Halbscheibe + plum Sonne als "warnte" Variante), bau ich's um.
  • PNG-Export. Nach deinem Approval rendere ich die 8 SVGs auf 24 / 48 / 72 px aus + lege sie als handoff/icons/png/{name}@{1,2,3}x.png ab.
