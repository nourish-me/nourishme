- [ ] **Scalable safety matching (stop substring false-positives)** · #P2 #safety
	The deterministic safety rules match by substring on short German stems, which over-matches compounds that are a different thing: kombu→Kombucha, muschel→Muschelnudeln, algen→Algenöl, each patched with a one-off exclusion. This whack-a-mole doesn't scale, and the false warnings erode trust in the safety layer (the most trust-sensitive surface). Needs a design pass (Explore). Options: (1) a golden food→verdict regression corpus (cheap, formalises intent, catches regressions); (2) compound-aware matching for German (kills classes of false positives, not one-offs); (3) the real fix — the parse-LLM emits a structured risk-category classification (raw seaweed vs refined oil supplement) that the deterministic layer gates on, with keywords as a backstop. Option 3 reintroduces LLM trust into safety (exactly what the deterministic layer guards against), so it needs a careful design + nutritionist sign-off. Lean (3) + (1) as the net. Conservative over-warning stays the default (over-warn > under-warn).
